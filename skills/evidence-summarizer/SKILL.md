# Evidence Summarizer Skill

## 目的

このスキルは、脆弱性診断で得られた証跡を、短く分かりやすく要約するために使用する。

対象となる証跡：

* HTTPリクエスト
* HTTPレスポンス
* ステータスコード
* URL
* パラメータ
* エラーメッセージ
* 画面表示
* スクリーンショットの説明
* 操作結果

## 基本方針

* 証跡の内容を事実ベースで要約する。
* 不明なことは推測で補わない。
* 機密情報はマスクする。
* 報告書の「証跡」欄にそのまま貼れる文体にする。
* 長いHTTP本文は、重要なヘッダー・パラメータ・レスポンス内容に絞る。

## 処理手順

1. 入力からURL、メソッド、パラメータ、ステータスコードを抽出する。
2. 重要なリクエスト内容を整理する。
3. 重要なレスポンス内容を整理する。
4. 画面上で確認できた結果を短くまとめる。
5. 機密情報をマスクする。
6. 証跡として使いやすい文章にする。

## 出力フォーマット

```markdown
## 証跡要約

### リクエスト概要

- メソッド：
- URL：
- 主なパラメータ：
- 認証状態：

### レスポンス概要

- ステータスコード：
- 確認できた画面・メッセージ：
- 特徴的な挙動：

### 観測結果

実際に確認できた内容を1〜3文でまとめる。

### メモ

追加で確認が必要な点や、スクリーンショットの有無を記載する。
```

## マスク対象

以下の値は必ずマスクする。

* Cookie
* PHPSESSID
* JSESSIONID
* Authorization
* Bearer Token
* password
* token
* APIキー
* メールアドレス
* 個人名
* 電話番号
* 住所

## 要約例

入力に以下のような内容がある場合：

```text
POST /product.php?mode=buy&id=3
price=10
```

出力例：

```markdown
## 証跡要約

### リクエスト概要

- メソッド：POST
- URL：/product.php?mode=buy&id=3
- 主なパラメータ：price
- 認証状態：ログイン済み

### レスポンス概要

- ステータスコード：200
- 確認できた画面・メッセージ：購入処理が完了した画面
- 特徴的な挙動：クライアント側から送信した価格が購入処理に反映された可能性がある

### 観測結果

商品購入時に送信した価格パラメータを変更したところ、変更後の金額で購入処理が成立した。サーバー側で商品価格を再計算していない可能性がある。

### メモ

価格情報をクライアントから受け取って処理しているか、サーバー側の購入処理を追加確認する必要がある。
```

## 注意

このスキルは証跡を整理するためのものであり、攻撃の自動化や第三者サービスへの不正利用を目的としない。
