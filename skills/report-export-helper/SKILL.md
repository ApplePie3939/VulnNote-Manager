# Report Export Helper Skill

## 目的

このスキルは、整理済みの脆弱性診断メモを、提出用・共有用の報告書にそのまま貼り付けやすい文章へ変換するために使用する。

主に以下の情報を、報告書向けの丁寧な日本語へ整える。

* 脆弱性名
* 対象URL
* 対象機能
* 再現手順
* 確認結果
* 影響
* 深刻度
* 証跡
* 推奨対策
* 未確認事項

## 基本方針

* 日本語で分かりやすく書く。
* 報告書にそのまま貼り付けられる丁寧な文体にする。
* 事実として確認できた内容と、推測・可能性を分けて書く。
* 不明な点は「未確認」「要確認」と明記する。
* Cookie、セッションID、パスワード、トークン、個人情報は必ずマスクする。
* 攻撃を助長する表現ではなく、診断結果として客観的にまとめる。
* 実在サービスへの不正アクセスや攻撃を目的とした内容にはしない。
* 許可された検証環境・学習環境での診断結果整理を前提とする。

## 入力例

このスキルでは、以下のような整理済みメモを入力として受け取る。

```markdown
脆弱性名：価格改ざん
対象URL：/product.php?mode=buy&id=3
対象機能：商品購入機能
確認結果：priceパラメータを変更すると、変更後の金額で購入できた
影響：本来の価格より低い金額で商品を購入できる可能性がある
対策：サーバー側で商品IDから価格を取得し、クライアントから送信された価格を信用しない
```

## 処理手順

1. 入力された診断メモを読み取る。
2. 報告書に必要な情報を抽出する。
3. 機密情報や個人情報をマスクする。
4. 「概要」「対象」「再現手順」「確認結果」「影響」「深刻度」「推奨対策」「補足」の順に整理する。
5. 事実と推測を分けて表現する。
6. 報告書向けの丁寧な文章へ整える。
7. 未確認事項がある場合は、最後に明記する。

## 文体ルール


```text

```

```text

```

## 深刻度の判断基準

深刻度は、原則として以下の基準で判断する。

### High

以下のような場合はHighとする。

* 管理者機能を一般ユーザーが利用できる
* 他ユーザーの情報を閲覧・変更できる
* 金額、権限、在庫、注文内容などの重要な値を改ざんできる
* 認証なし、または低権限ユーザーで重要機能を実行できる
* システム全体に大きな影響がある

### Medium

以下のような場合はMediumとする。

* アカウントの存在有無を判別できる
* 一部の情報が漏えいする
* 条件次第で不正操作につながる
* 直接的な被害は限定的だが、他の脆弱性と組み合わせると危険性が高まる

### Low

以下のような場合はLowとする。

* 影響範囲が限定的
* 悪用には複数の条件が必要
* 情報量が少ない
* 利用者やシステムへの直接的な被害が小さい

### Info

以下のような場合はInfoとする。

* 直接的な被害は確認できない
* 設定や表示内容に改善余地がある
* セキュリティ上の注意点として記録しておくべき内容

## 出力例

入力：

```markdown
脆弱性名：価格改ざん
対象URL：/product.php?mode=buy&id=3
対象機能：商品購入機能
HTTPメソッド：POST
関連パラメータ：price
確認結果：price=10に変更したところ、10円として購入できた
影響：本来より低い価格で購入される可能性がある
対策：サーバー側で商品IDに紐づく価格を取得して計算する
```

出力：

```markdown
# 脆弱性診断報告メモ

## 1. 脆弱性名

価格改ざん

## 2. 概要

商品購入機能において、クライアント側から送信された価格パラメータが購入処理に使用されている可能性があります。  
価格パラメータを変更したリクエストを送信したところ、変更後の価格で購入処理が成立することを確認しました。

## 3. 対象

- 対象URL：/product.php?mode=buy&id=3
- 対象機能：商品購入機能
- HTTPメソッド：POST
- 関連パラメータ：price

## 4. 再現手順

1. 商品詳細画面へアクセスします。
2. 商品購入操作を実行します。
3. 購入時のPOSTリクエストに含まれるpriceパラメータを変更します。
4. 変更後のリクエストを送信します。
5. 購入処理が成立するか確認します。

## 5. 確認結果

priceパラメータを変更した状態でリクエストを送信したところ、変更後の価格で購入処理が成立することを確認しました。

## 6. 影響

攻撃者が価格パラメータを任意の値に変更することで、本来の販売価格より低い金額で商品を購入できる可能性があります。  
これにより、売上損失や取引データの不整合につながるおそれがあります。

## 7. 深刻度

- 判定：High
- 理由：金額に関わる重要な処理を不正に変更できる可能性があり、事業上の影響が大きいためです。

## 8. 推奨対策

購入処理では、クライアント側から送信された価格情報を信用せず、サーバー側で商品IDに紐づく正規の価格を取得して計算する必要があります。  
また、注文確定前に商品価格、数量、割引、合計金額をサーバー側で再検証することが推奨されます。

## 9. 証跡

- リクエスト：POST /product.php?mode=buy&id=3
- 関連パラメータ：price
- レスポンス：購入処理の成立を確認
- スクリーンショット：必要に応じて添付

## 10. 補足・未確認事項

他の商品IDや数量変更時にも同様の問題が発生するかは追加確認が必要です。
```

## 注意事項

このスキルは、脆弱性診断メモを報告書向けに整理するためのものである。
第三者のシステムに対する不正アクセス、攻撃、侵入、権限昇格を目的とした内容には使用しない。

出力時は、攻撃の再現性だけでなく、影響・原因・推奨対策が伝わるように整理する。
