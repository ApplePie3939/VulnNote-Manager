# Project Overview

このリポジトリでは、Pythonを使用して「VulnNote Manager」を開発します。

VulnNote Managerは、脆弱性診断中に記録した断片的なメモを整理し、再確認しやすい診断記録や報告書の下書きへ変換するためのツールです。

主な目的は次のとおりです。

- 診断メモを案件・対象・脆弱性ごとに整理する
- 再現手順、証跡、影響、対策を一貫した形式で記録する
- 記録漏れや表現のばらつきを減らす
- MarkdownやCSVなどの形式へ安全に出力する
- AIを整理・文章作成の補助に利用する
- 未確認事項を事実として補完しない

## Technology Policy

- 使用言語はPythonとする
- Python 3.12以降を第一候補とする
- 標準ライブラリで十分な場合は、不要な外部ライブラリを追加しない

既存の設定ファイルや依存関係がある場合は、それらを優先してください。ライブラリを新しく追加する場合は、目的と必要性を明確にしてください。

## Working Language

- ユーザーへの説明、仕様書、設計書、診断レポートは原則として日本語で記述する
- Pythonの変数名、関数名、クラス名、モジュール名は英語で記述する
- 専門用語には必要に応じて短い説明を添える
- 初学者でも変更理由を追えるよう、具体的なファイル名と操作内容を示す
- エラーメッセージは、利用者が原因と対処方法を判断できる表現にする
